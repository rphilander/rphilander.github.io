Proteus

This is a description of Proteus. You may also watch a video or peruse the repo.

Proteus is a query builder with a wholly different user experience from other query builders — you “build” queries by entering keywords. The user experience is similar to search engines such as Google or Bing. As you enter your keywords, relevant queries are displayed. More specifically, English sentences describing relevant queries are displayed. You click on an English description to execute the associated query.

As a concrete example, suppose the target database contains NFL scores and statistics and I, the end user, want to find out which quarterback threw the most touchdown passes in 2011. I might provide as input: “most touchdowns 2011”. The search results would include a number of queries related to those keywords, such as: which team scored the most touchdowns in 2011, which receiver caught the most touchdown passes in 2011, which quarterback threw the most touchdown passes in 2011, and so forth. I click on the appropriate query and I get the desired information.

While Proteus creates the illusion of searching through a multitude of already-existing queries, the implementation is quite different. For even a relatively simple database, the number of potentially interesting queries is immense. It would be impractical to generate and index that many web pages. Instead, the user’s keywords are scanned to identify tokens and then, using a set of templates, those tokens are combined to form queries. In this context, tokens are words or short phrases which have some meaning in terms of the target database. For the hypothetical NFL database, tokens would include the names of teams, the names of players, the years in which football games were played, such terminology as “touchdown” and “field goal”, and so forth. The templates allow those tokens to be combined in two different ways at the same time: once to generate a SQL statement, and once to generate an human-readable description (in English) of what that SQL statement does. The English descriptions are displayed to the user and, when the user selects a particular query for execution, the corresponding SQL is run against the target database. The results are then displayed to the user.

The tokens and templates are created when Proteus is configured for a particular database. Note that most tokens exist in the database itself, for instance the names of teams in the NFL example, and can therefore be defined implicitly in SQL (e.g. SELECT DISTINCT team_name FROM teams;). The templates must be crafted with some care — they define the set of queries supported by an instance of Proteus and should meet the needs of the end users.

Proteus exists because the extremely useful skill of building queries is hard and complex. In essence, it is a kind of programming. The person building the query must think simultaneously in two domains: the real world, in which the need for particular information exists, and the computer world, in which very precise and unambiguous code must be crafted. To be successful the code must not only run without errors, but address the real-world needs that motivated it. Since relatively few people have this skill, most people must either rely on programmers, which is expensive, or use a query building tool. Existing query building tools are quite powerful, but I have personally found them limited in the range of queries they can support. With existing tools you are essentially building a query using buttons and sliders and pull-down menus and the like, and there is a limit to how many buttons and sliders and pull-down menus can be presented to the user before the visual experience becomes overwhelming. Proteus, in contrast, leverages the combinatorial expressiveness of the SQL, but wraps it in a friendly natural-language package to make that expressiveness accessible to the non-programmer. To think of it another way: the user is able to drive the difficult and error-prone process of building queries, through the very familiar and intuitive process of entering keywords and reading the English sentences.
